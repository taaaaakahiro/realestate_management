# PropFolio 要件定義

不動産投資管理アプリ「PropFolio」の開発要件を整理したドキュメント。
（操作手順は [MANUAL.md](MANUAL.md)、入力規則・計算式は [SPEC.md](SPEC.md) を参照）

---

## 1. 背景・目的

- 不動産投資において、**購入時の原価・日々の支出・家賃収入・融資**を一元管理し、
  投資の**回収率（取得原価をどれだけ家賃で回収できたか）**を見える化する。
- 取得前（検討中）の物件は、**想定家賃と目標利回りから取得可能な価格を逆算**し、購入判断を支援する。

## 2. スコープ

- 個人投資家向けの**シングルユーザー・クライアント完結**のWebアプリ（モック）。
- データはブラウザ内（localStorage）に保存。サーバー・認証・共有は対象外。

## 3. 機能要件

### 3.1 物件管理
- 物件の登録・編集・削除。
- 状態（**取得前 / 保有中 / 売却済み**）の管理と相互切替。一覧で状態フィルター。
- 物件一覧は**都道府県・市区町村でフィルター**でき、**並び替え**（地域別グループ表示／回収率／取得原価／取得日／物件名）に対応。地域別を選ぶと都道府県→市区町村でグループ表示（住所から自動判定）。
- ポートフォリオ集計（回収率・サマリー）は**保有中のみ**を対象とする。
- 郵便番号から住所を自動補完。住所クリックで Google マップ検索。

### 3.2 取得原価・取得シミュレーション
- 物件価格を**評価額比で土地・建物に按分**（端数は土地）。
- **不動産取得税・固定資産税精算金・印紙代**を自動計算（印紙代は印紙税法に基づく）。
- **登記費用（登録免許税＋司法書士報酬）は手入力**（自動計算しない）。
- **取得原価 = 物件価格＋不動産取得税＋固定資産税精算金＋登記費用**。
- 取得前は **目標利回り → 投資総額 → 経費控除 → 物件価格** を逆算（経費＝取得税＋精算金＋登記費用＋仲介＋印紙）。

### 3.3 融資（ローン）
- 借入銀行（**全国の地銀・第二地銀・信用金庫・信用組合等を検索選択**／自由入力）。
- 借入元本・当初金利・返済期間・返済方式（**元利均等／元金均等**）・返済開始日・**手出し金額（自己資金）**。
- 毎月返済を**元本・利息に自動分解**。**変動金利**（途中変更）に対応。
- **繰り上げ返済**（**期間短縮型／返済額軽減型**）に対応。残債・利息・完済予定に反映。

### 3.4 収支管理
- 収入・支出の登録・編集・削除。科目体系を用意。
- **複数件の一括登録**（3〜10件、行の追加・削除可）。
- ローン返済は融資情報から元本・利息へ自動分解（単票）。
- 取引明細：区分・年・月・**科目**でフィルター、ページネーション、**CSV／PDF エクスポート**。

### 3.5 分析・可視化
- ダッシュボード：投下資本回収率・取得原価・損益・資産構成（種別別円グラフ）・売却済みの実現損益。
- 物件詳細：回収率、推移チャート、表面/実質利回り、月次CF。
- **回収予測**：直近（**1か月／3か月／半年／1年**から選択）の収入ペースで、取得原価の回収まで**あと何年何か月**かを試算。取得日から未経過の期間は選択不可。
- 売却：売却価格・経費・売却日から**譲渡損益・実現損益**を算出。
- 収支・損益は**色分け**（収入=緑／支出・マイナス=赤）。

## 4. 非機能要件

- **依存を薄く**：ランタイム依存は react / react-dom のみ（セキュリティ・保守性重視。Next.js から移行）。
- ルーティングは外部ライブラリ不使用の自前ハッシュルーター。
- **コンポーネント化**して修正の影響範囲を小さく保つ（機能単位の feature 構成、純粋関数で計算分離）。
- 入力は**バリデーション**で不正値を排除（文字列・数値・日付を明確化、3桁カンマ表示）。
- レスポンシブ対応（モバイルメニュー等）。
- GitHub Pages で静的公開（`npm run build` → `dist`）。

## 5. 制約・前提

- 税率・特例は概算の既定値（中古住宅控除・住宅用地の税額軽減等は未適用）。
- 銀行一覧は主要機関を広く収録するが網羅ではない（不足は自由入力）。
- データは端末のブラウザ内のみ。クリアすると初期サンプルに戻る。

## 6. 技術スタック

- React 19 / Vite / TypeScript / Tailwind CSS v4
- データ層：`src/data/store.ts`（localStorage、`useStore`）
- 計算：純粋関数（`features/analytics`・`features/loan`・`features/simulation`）
